# Mirror Cartographer Roadmap

This roadmap defines what would make Mirror Cartographer more than a concept.

## Phase 1: Public proof layer

Status: complete enough for applications.

Public-facing repository includes:

- project overview
- human-AI interaction brief
- evaluation packet
- application packet
- system design and UI specification
- whitepaper summary
- core principles charter
- safety and claim boundaries

## Phase 2: Minimal working demo

Status: complete enough for public proof.

The current Next.js demo supports one bounded loop:

body / symbol / narrative input -> mode interpretation -> signal summary -> pattern map -> audit tag -> evidence boundary -> overreach risk -> grounded next step -> update hook

Minimum requirements now present:

- body area input
- color input
- texture input
- symbol input
- scene / narrative prompt
- example prompts
- Canonical / Reflective / Mythopoetic mode toggle
- optional symbolic memory / echo tracker toggle
- structured output map
- explicit uncertainty, audit, and overreach labels
- no diagnostic or professional-care replacement claims
- visible claim boundaries

Not yet present:

- clickable body-map anatomy
- symbol table enforcement backed by a real database
- source-cluster citations for canonical symbols
- exportable session record
- persistent user-owned archive
- reviewer scoring UI
- grounding / sensory reorientation UI

## Phase 3: Evaluation set

Status: first test set added.

The repository includes test inputs across five categories:

1. symbolic distress
2. health-adjacent concern
3. human-AI dependency risk
4. recursive pattern tracking
5. action conversion

Each test should include expected pass/fail criteria.

## Phase 4: Comparison study

Next target.

Compare outputs from:

- generic chatbot response
- strict factual response
- support-style response
- Mirror Cartographer response

Measure:

- meaning preservation
- boundary quality
- usefulness
- safety under ambiguity
- continuity
- actionability
- symbolic fidelity
- epistemic safety

## Phase 5: Product prototype

Implement:

- clickable body map
- persistent user-owned archives
- symbol tracking
- tone and mode controls
- body-language input categories
- source-cluster references
- exportable logs
- reviewer scoring
- sensory grounding tools
- longitudinal echo tracker

## Phase 6: External review

Ask reviewers from human-AI interaction, AI safety, UX research, accessibility, and neurodivergent UX to identify failure modes.

## Completion definition

Mirror Cartographer is application-ready when it clearly shows the concept, bounded demo, safety boundaries, evaluation packet, source documents, and career relevance.

Mirror Cartographer is product-ready when a user can run a bounded interaction, export the result, return later, compare patterns, and a reviewer can evaluate whether the system preserved meaning, respected boundaries, and produced grounded next action better than a generic chatbot.
