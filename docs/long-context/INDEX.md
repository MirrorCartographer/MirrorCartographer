## Core flow recovered from older notes

Older Mirror Cartographer specs used the flow:

ENTRY -> FIELD -> RECURSION -> RETURN

The current public-facing version translates that into:

input -> bounded reflection -> feedback -> correction -> safer next output -> saved trace

Both flows are related. The newer version is safer, more product-facing, and more testable.
