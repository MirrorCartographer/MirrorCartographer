# Upload Batch Face/Signal Reading

Revision note:

- Status: public-safe synthesis of a current upload batch.
- Reason: created after the user asked how to understand face value versus deeper meaning, then uploaded sigil images and multiple Mirror Cartographer files.
- Source: current upload batch including a core sigil image, a generated mirror-map image, cognitive infrastructure/profile documents, transition/build plans, ritual/protection documents, and public-facing MC artifacts.

## Core method

Read every artifact through four layers:

1. Face value
2. Signal value
3. Evidence value
4. Action value

Do not skip the surface.

Do not mistake symbolic force for proof.

Do not erase symbolic force just because it is not proof.

## Artifact class rules

### Visual primitive

Can guide interface design.

Cannot prove the system works.

### Concept art

Can communicate atmosphere and public curiosity.

Cannot replace implementation.

### Cognitive profile document

Can inform profile schema.

Cannot diagnose or permanently define a person.

### Build checklist

Can prove planning history.

Cannot prove completion.

### Ritual worksheet

Can become symbolic UI.

Cannot become medical authority.

### Protection protocol

Can express authorship and ethics.

May need legal translation for enforceability.

### Whitepaper

Can propose theory and architecture.

Needs evidence labels and prior-art work.

### Public proof artifact

Can communicate value.

Needs source-backed examples and external validation.
