# Private Core and Public Echo Engine

Revision note:

- Status: source-derived and AI-organized architecture note.
- Reason: created to preserve the older distinction between a private personal Mirror Cartographer engine and a public generalized tool.
- Source: prior long-form conversation exports and old chat logs describing Core Entity / Origin Intelligence and Echo Engine / Public System.
- Boundary: this is an architecture concept and privacy boundary, not a claim that both systems are fully built.

## Core distinction

Older Mirror Cartographer conversations separated the system into two roles:

1. Private Core Entity / Origin Intelligence
2. Public Echo Engine / Public System

This distinction matters because not all inner logic should become public product logic.

## 1. Private Core Entity / Origin Intelligence

Purpose:

- preserve the creator's origin logic
- work with personal body, symbol, memory, thought, and recursive pattern
- remain private, protected, and undiluted by mass users

Data types:

- personal logs
- body signals
- symbolic laws
- private rituals
- archetypes
- health and animal-care observations
- language patterns
- AI interaction history

Integrity constraint:

The private core should not be trained away from the creator's signal by mass-user averages.

It is the seed crystal.

## 2. Public Echo Engine / Public System

Purpose:

- translate the architecture into a generalized tool for others
- provide clean templates
- help users map somatic-symbolic-emotional patterns
- preserve user agency and consent
- support public research, evaluation, and product use

Data types:

- user-controlled input
- optional session memory
- public symbol tables
- source-backed references
- exportable traces
- correction history

Integrity constraint:

The public system may be derived from the origin architecture, but it should be sandboxed from private data.

## Why this matters

This split solves a major authorship and safety problem.

The private system can hold deep personal continuity.

The public system must be safer, bounded, more general, consent-based, and testable.

## Product translation

Public Mirror Cartographer should include:

- no private creator data
- clear mode boundaries
- export and delete controls
- visible claim status
- visible source status
- user correction
- no diagnostic authority
- no claim that symbolic resonance equals truth
