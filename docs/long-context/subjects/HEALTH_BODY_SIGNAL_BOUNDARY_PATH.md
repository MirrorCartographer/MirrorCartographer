## Why this belongs in Mirror Cartographer

A major long-context thread involved felt body signals that were hard to explain linearly.

Mirror Cartographer's role is not to name causes.

Its role is to help organize:

- what was noticed
- where it was noticed
- when it happened
- what changed it
- what repeated
- what needs outside review
- what should not be concluded yet

## Public-safe scope

This public file does not include private records, exact timelines, lab-style markers, appointment details, or personal case specifics.

Specific records should stay private unless intentionally anonymized and reviewed for public release.

## Core principle

Symbol can organize attention.

Symbol cannot prove cause.

## Useful structure

### Observation

What happened, described as concretely as possible.

### Location

Where the sensation or change appeared.

### Timing

When it happened, how long it lasted, and what came before it.

### Context

Sleep, food, heat, movement, stress, routine, travel, posture, cycle timing, or environment.

### Change

What improved, worsened, moved, disappeared, or stayed.

### Boundary

What needs outside review, urgent support, or no interpretation yet.

## Product translation

Mirror Cartographer body-signal mode should produce:

- observation record
- body map
- timeline
- uncertainty label
- outside-review boundary
- question list
- what not to conclude
- optional symbolic layer clearly marked as symbolic

## Old-chat insight retained

The user often described body and perception through symbolic or spatial language.

The system should not flatten that language into generic checklists, but it also must not inflate it into certainty.
