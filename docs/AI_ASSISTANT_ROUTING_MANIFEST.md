# AI Assistant Routing Manifest

Revision note:

- Status: public-safe AI navigation manifest.
- Reason: created so future AI assistants can work on this repository without flattening the project, exposing private details, or confusing research hypotheses with proven claims.
- Source: long-context organization work, privacy boundary, school-subject layer, research provocation index, GitHub implementation files, and project-management direction.
- Boundary: this is a routing guide for assistants. It does not give permission to publish private source details.

## First rule

Do not treat the repository as one essay.

Treat it as a routed knowledge system.

## Before adding anything

Ask:

1. Is this public-safe?
2. Is it source-derived, AI-synthesized, or implemented?
3. Is it a method, research question, education layer, private-risk layer, or code change?
4. Does it need a revision note?
5. Does it make a claim that needs a boundary?
6. Which folder owns it?

## Required labels

Whenever useful, label material as:

- Observation
- User report
- Source-derived synthesis
- Symbolic hypothesis
- Product requirement
- Research provocation
- Implementation status
- Boundary / non-claim

## Privacy behavior

Abstract private specifics.

Use public-safe categories.

Do not publish raw chat exports, private health details, specific household animal case details, account data, relationship details, exact addresses, or unnecessary identifiers.

## Claim behavior

Do not write as if MC is already proven.

Use:

- testable hypothesis
- product requirement
- source-derived pattern
- public-safe abstraction
- needs evaluation
- not yet implemented

Avoid:

- proven breakthrough
- diagnostic tool
- therapeutic authority
- objective truth engine
- oracle
- replacement for professional care

## Good assistant behavior

A good assistant working on this repo should:

- preserve source signal
- organize instead of delete
- revise with reasons
- cite or register source paths
- protect private details
- distinguish public tool from private origin logic
- convert critique into implementation
- make search terms explicit
- keep artifacts useful for both humans and AI

## Final instruction

Mirror Cartographer is strongest when the assistant does not pretend to know.

The assistant should help the user and future readers preserve, route, test, correct, and build from signals without turning those signals into false certainty.
