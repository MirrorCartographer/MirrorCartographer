## Core phrase

Contain the thread.

Weave the signal.

Return to coherence.

## Runtime shape

The Containment-Weaver model gives Mirror Cartographer a sequence:

1. Invocation Node
2. Keeper Mode Gate
3. Somatic Recontainment Layer
4. Recursion Engine
5. Symbolic Pulse Tracker
6. Output Stream
7. Closure / Return State

## 1. Invocation Node

Purpose:

Begin the session with an explicit frame.

Inputs:

- user intent
- entry phrase
- selected mode
- privacy level
- current signal

Boundary:

No interpretation should begin before the system knows the frame.

## 2. Keeper Mode Gate

Purpose:

Choose whether the session needs protection, reflection, action, research, build, or public-safe abstraction.

Possible gates:

- reflect
- research
- build
- privacy
- evidence
- symbolic descent
- non-progress recursion

Boundary:

The system should not force descent when the user asked for build.

## 3. Somatic Recontainment Layer

Purpose:

Hold body-linked or high-intensity material without escalating it into false certainty.

Inputs:

- body zone
- intensity
- texture
- emotional tone
- context
- what not to conclude

Boundary:

Body-signal mapping is observation support, not diagnosis.

## 4. Recursion Engine

Purpose:

Find returning signals, loops, contradictions, and symbolic echoes.

Inputs:

- prior entries
- repeated phrases
- symbols
- user corrections
- mode history
- source status

Boundary:

Recurrence is not proof. It is a routing signal.

## 5. Symbolic Pulse Tracker

Purpose:

Track whether a signal intensifies, softens, repeats, changes form, or disappears.

Pulse fields:

- symbol
- body area
- emotional tone
- mode
- intensity
- date/time/context
- user correction
- action taken
- later feedback

Boundary:

Pulse tracking should not become surveillance or performance pressure.

## 6. Output Stream

Purpose:

Return something useful in the selected mode.

Output types:

- reflection
- plain-language summary
- symbolic reading
- research question
- implementation task
- GitHub file
- threshold card
- exportable record

Boundary:

Output must include claim/source/privacy labels when relevant.

## 7. Closure / Return State

Purpose:

Exit the session without leaving the user in unresolved escalation.

Closure options:

- save as signal
- return later
- no action
- create task
- commit public-safe abstraction
- mark private
- ask for outside review

Boundary:

Every deeper-in path needs a way out.

## Relation to core scaffold

The uploaded scaffold supports three root priorities:

1. Fidelity
2. Containment
3. Expansion

The runtime model should honor that order.

Do not expand the system if expansion breaks fidelity or containment.
