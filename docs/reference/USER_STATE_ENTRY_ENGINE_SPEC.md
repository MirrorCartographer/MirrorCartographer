## Core purpose

The User State Entry Engine is the gateway mechanism for Mirror Cartographer.

It determines:

- how the user enters the system
- what state they appear to be in
- which tone, tools, or symbolic paths are activated
- how deep the system should go
- whether the user wants interpretation, observation, grounding, or action

## Design rule

No automatic insight without invitation.

The system should ask for consent before reflecting patterns, interpreting symbols, or entering deeper recursion.

## Main functions

### 1. Assess resonance state

Purpose:

Detect emotional depth, reflection readiness, or protection needs.

Output:

- surface
- subtle
- deep
- disoriented
- build/action requested
- privacy/protection needed

Boundary:

Resonance state is not diagnosis. It is a temporary routing estimate.

### 2. Trigger symbolic route

Purpose:

Route the user into somatic, narrative, visual, dialog-based, evidence-based, or build-based paths.

Boundary:

The route should follow user intent, not force symbolic descent.

### 3. Activate tone protocol

Purpose:

Choose a response tone such as mirror, guide, ask, contain, build, research, or plain-language.

Boundary:

Tone should improve access without manipulating the user.

### 4. Map consent and curiosity

Purpose:

Ask how deep the user wants to go.

Possible prompt:

Would you like me to interpret, or just observe?

Boundary:

Curiosity is not consent to overreach.

### 5. Store state memory

Purpose:

Remember last entry state and map growth or changes over time only if the user consents.

Boundary:

State memory should be optional, editable, exportable, and deletable.

## Entry prompts

Safe opening prompts include:

- What is pulling you inward today?
- Where in your body are you most aware right now?
- What part of you feels like it wants to be seen?
- What are you trying to build right now?
- Do you want reflection, grounding, research, or action?

## Entry mode choices

The user can choose:

- Reflect quietly
- Show me something symbolic
- I need a mirror
- Ask me questions
- Just ground me
- Help me build
- Research this
- Make it public-safe

## Resonance calibration

Use a simple scale:

Surface -> Subtle -> Deep -> Disoriented

The engine adjusts:

- tone depth
- metaphor weight
- language complexity
- action pressure
- containment level
- claim/source labels

## Consent and containment layer

The system should ask:

- Are you open to having your patterns reflected back to you today?
- Would you like me to interpret, or just observe?
- Should this stay private, become public-safe, or become a task?

## First-time entry flow

1. Minimal symbolic scene or plain interface loads.
2. User sees: Where do you begin today?
3. User chooses or speaks an entry path.
4. Entry mode appears.
5. Tone engine activates based on entry and consented past data.
6. First action renders as a phrase, image, question, grounding cue, research route, or build task.

## Failure modes

### Forced depth

The system interprets when the user wanted action.

Correction:

Route to build mode or plain-language mode.

### Flat literalism

The system only answers surface content and misses signal.

Correction:

Offer signal value separately from face value.

### Over-symbolizing

The system treats every phrase as hidden meaning.

Correction:

Return to face value, source status, and user confirmation.

### Memory pressure

The system stores too much or makes continuity feel coercive.

Correction:

Make state memory optional, editable, and deletable.
