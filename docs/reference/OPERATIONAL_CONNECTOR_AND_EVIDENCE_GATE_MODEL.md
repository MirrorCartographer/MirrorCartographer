## Core correction

The missing connector is execution.

Mirror Cartographer needs a working connector map, not only a conceptual thesis.

## Operational connector types

### 1. Proof lanes

Connects:

Meaning to evidence domains.

Works by:

Separating lanes such as software, research, art, education, privacy, and public proof.

Failure if absent:

Metaphor leaks into factual claims.

### 2. Failure ledger

Connects:

Wrong output to next build.

Works by:

Logging failure class and creating the roadmap.

Failure if absent:

Failure stays shame, noise, or forgotten frustration.

### 3. Archive index

Connects:

Fragments to continuity.

Works by:

Storing path, state, source, and next action.

Failure if absent:

Ghost files accumulate.

### 4. Evidence gate

Connects:

Claim to reality contact.

Works by:

Requiring score, diagnostic, scope, response, test, source, or reproducible artifact depending on lane.

Failure if absent:

Coherence pretends to be proof.

### 5. Affordance taxonomy

Connects:

Object, media, place, or idea to possible human action.

Works by:

Sorting by what becomes possible, not only by topic.

Failure if absent:

Topic sorting misses usefulness.

### 6. Signal reflection mode

Connects:

Expression to non-praise reflection.

Works by:

Naming pattern without identity inflation.

Failure if absent:

The system falls into flattery loops.

## Ghost / gremlin / symbolic / non-symbolic

### Ghost

Unresolved persistence: something still influences the system after its source, file, or context is absent.

### Gremlin

Execution destabilizer: something actively breaks routing, proof, memory, or update behavior.

### Symbolic

Names, rituals, glyphs, images, and archetypes that function as meaning engines.

Boundary:

Symbolic material is not evidence by itself.

### Non-symbolic

Scores, logs, code execution, diagnostics, official environments, real users, reproducible artifacts, and actual file existence.

Boundary:

Non-symbolic evidence can still be incomplete or misinterpreted.

## Discovered versus generated structure

### Discovered structure

Resists preference.

Examples:

- benchmark scores
- code execution
- diagnostics
- actual file existence
- real user behavior
- deployment status
- external review

### Generated structure

Is interpretation.

Examples:

- symbolism
- names
- ritual logic
- story arcs
- aesthetic meaning
- archetypal mapping
- speculative theory

## Collapse risk

Collapse happens when generated structure is treated as discovered evidence.

Mirror Cartographer should value both, but route them differently.

## Lane fields

Every lane should include:

- lane
- current state
- source file
- proof standard
- next action
- ghost flag
- gremlin flag

## Update cycle

Use the cycle:

search -> cite -> classify -> connect -> test -> artifact -> index -> next action

## Still-unfound data category

The system should keep a visible list of still-unfound data.

Examples:

- raw source exports not yet available
- final runtime specs not yet implemented
- claims not yet externally tested
- audience/user data not yet collected
- generated artifacts not yet verified as rendered
- status ledgers not yet reconciled
