## How to read this file

Each item is a research thread, not a conclusion.

Use each provocation to ask:

- What is the observation?
- Why might it matter?
- What would make it false?
- What system behavior would test it?
- What discipline would care?

## Top-level claim

The most interesting thing in Mirror Cartographer is not that it uses symbols.

The interesting thing is that it treats symbolic, somatic, emotional, linguistic, visual, and interaction-pattern data as low-confidence signals that can be preserved, weighted, corrected, and converted into action without pretending they are proof.

## Research provocations

### 1. Everything can carry signal, but signal is not equal

Information should be weighted, not flattened into true/false.

Test: Can users benefit from seeing signal confidence levels instead of binary interpretations?

Disciplines: human-AI interaction, cognitive science, UX research, AI safety, semiotics.

### 2. Typos can be interface data

Typos may reflect speed, emotion, motor habit, fatigue, dialect, device friction, or attention drift.

Test: Does preserving typo patterns improve user understanding without increasing overinterpretation?

Disciplines: linguistics, accessibility, HCI, cognitive science.

### 3. Nonstandard language may be precision, not error

Slang, dialect, compressed wording, and fragments can carry exact meaning that formal correction erases.

Test: Compare standard-corrected responses against responses that preserve user language force.

Disciplines: sociolinguistics, accessibility, UX writing, AI alignment.

### 4. A reflection can be eloquent and still not fit

Response quality must include fit-feedback, not only fluency.

Test: Ask users to rate whether a response fits, grounds, misses, or overreaches.

Disciplines: HAI, UX research, evaluation science.

### 5. Symbolic resonance is not evidence

A symbol feeling meaningful should never be converted into factual certainty.

Test: Can the system keep resonance separate from proof across repeated sessions?

Disciplines: AI safety, epistemology, clinical-adjacent UX, semiotics.

### 6. The body may signal before language stabilizes

Somatic descriptions can be treated as user reports, not diagnoses.

Test: Does body-symbol mapping help users organize observations without increasing false medical certainty?

Disciplines: HCI, embodied cognition, health UX, AI safety.

### 7. A repeated question may be a loop detector

Repetition can indicate unresolved proof need, action blockage, or missing category.

Test: Can the system detect recurrence without pathologizing repetition?

Disciplines: cognitive science, interaction design, AI behavior analysis.

### 8. The product is the loop, not the answer

A one-time answer is weaker than input -> reflection -> feedback -> correction -> saved trace.

Test: Compare one-shot chatbot output to iterative correction-based mapping.

Disciplines: HAI, AI evaluation, longitudinal UX.

### 9. A mirror becomes safer when it can be corrected

A non-correctable reflective system risks becoming an oracle.

Test: Does user correction reduce overtrust and improve action clarity?

Disciplines: AI safety, model behavior, UX research.

### 10. Claim status should be visible

Every output should identify whether it is observation, user report, symbolic hypothesis, fact, next step, or unknown.

Test: Does visible claim status reduce confusion and false confidence?

Disciplines: epistemology, AI safety, UX, education.

### 11. Source status should be visible

Symbolic outputs should show whether they are source-backed, user-backed, speculative, or unavailable.

Test: Does source status improve epistemic safety?

Disciplines: information science, AI safety, research methods.

### 12. High-intensity language needs de-escalation paths

Powerful language can energize or destabilize.

Test: Does offering neutral/scientific mode reduce emotional overactivation?

Disciplines: clinical-adjacent UX, HAI, accessibility.

### 13. Not all distress is immediate

Some adaptation needs emerge later, not at first contact.

Test: Does delayed-review mapping catch useful changes missed by immediate reflection?

Disciplines: rehabilitation psychology, longitudinal UX, HCI.

### 14. Not all distress is universal

A system should not assume every rupture is traumatic.

Test: Does neutral option improve user trust compared with automatic intensity framing?

Disciplines: psychology, trauma-informed design, ethics.

### 15. Ceiling breaking is a distinct signal class

Some moments are not loops or holes. They are upper-bound ruptures: a system hits its ceiling, breaks containment, and something above the normal frame enters awareness.

Possible forms:

- breakthrough
- overload
- opportunity
- exposure
- spiritual or symbolic expansion
- collapse of old capacity
- sudden access to a higher-demand environment

Test: Can users distinguish ceiling-breaking signals from repetition loops, crisis signals, and ordinary ambition?

Disciplines: creativity research, HAI, psychology, systems theory, design research.

### 16. Ceiling breaking requires containment, not only encouragement

When the ceiling breaks, the right response is not always hype. It may need containment, pacing, scaffolding, and translation.

Test: Does a containment-first response reduce overwhelm while preserving creative expansion?

Disciplines: creativity studies, mental-health-adjacent UX, AI safety.

### 17. The opposite of a ceiling is not freedom; it may be exposure

Breaking a ceiling can expose the user to more input than they can organize.

Test: Can an interface detect when expansion needs grounding instead of acceleration?

Disciplines: sensory design, HCI, nervous-system-informed UX.

### 18. The hole and the ceiling form a pair

The hole is a repeated downward trap. The ceiling is an upper limit that blocks or suddenly breaks.

Test: Does mapping both patterns help users identify whether they need avoidance repair, capacity building, or containment?

Disciplines: systems theory, interaction design, coaching research.

### 19. Prosthetic interface is a useful model for symbolic tools

A symbolic tool should restore a control pathway, not admire the wound.

Test: Can users identify what agency/function the tool restored after a session?

Disciplines: rehabilitation design, HCI, embodied cognition.

### 20. Fit is a safety property

If an AI reflection does not fit the user, the system should downgrade confidence.

Test: Does fit-feedback reduce overtrust and improve subsequent outputs?

Disciplines: AI safety, UX research, personalization.

### 21. Maintenance is part of meaning

A system that works once but cannot be maintained over time is not a stable interface.

Test: Do session archives and follow-up reviews improve practical usefulness?

Disciplines: HCI, personal informatics, longitudinal AI systems.

### 22. The first signal may be visual

Some users may think in images before words.

Test: Do visual-symbol inputs improve clarity for users who struggle with linear verbalization?

Disciplines: visual cognition, accessibility, neurodivergent UX.

### 23. Body maps are translation surfaces

A body map can help convert vague internal experience into location, color, texture, and symbol.

Test: Does a clickable body map produce better user recall or more accurate self-report than text alone?

Disciplines: HCI, somatic psychology, data visualization.

### 24. A symbol can be user-backed without being universal

A user's personal association may matter even if it is not culturally canonical.

Test: Does separating user-backed and source-backed meaning improve trust?

Disciplines: semiotics, anthropology, personalization research.

### 25. Mythopoetic mode needs explicit borders

Creative, poetic, speculative outputs can be useful if labeled as such.

Test: Does explicit mode labeling let users enjoy symbolic richness without mistaking it for authority?

Disciplines: AI safety, creative tools, UX writing.

### 26. The system should preserve contradiction

Contradictions may be the map, not an error.

Test: Does preserving contradiction improve later insight compared with forcing immediate resolution?

Disciplines: cognitive science, psychotherapy research, HCI.

### 27. User correction is data ownership

When users correct a system, they are asserting ownership over interpretation.

Test: Does correction history improve agency and trust?

Disciplines: data ethics, HCI, AI governance.

### 28. Searchability changes usefulness

A messy archive becomes more valuable when indexed by subject, signal type, boundary, and evidence status.

Test: Do researcher-facing indexes improve project comprehension and reuse?

Disciplines: information architecture, knowledge management.

### 29. Public proof requires public boundaries

A project becomes more credible when it clearly says what it does not prove.

Test: Do explicit non-claims improve researcher trust?

Disciplines: science communication, AI ethics, product design.

### 30. Aesthetic force can hide weak mechanics

Beautiful language can make a system feel more complete than it is.

Test: Does hostile audit improve perceived credibility among technical reviewers?

Disciplines: design criticism, HAI, research communication.

### 31. The most useful critique is implementation-facing

Critique is strongest when it becomes code, test, or product behavior.

Test: Compare abstract critique with critique converted into UI requirements.

Disciplines: engineering management, design research, HCI.

### 32. A project can be co-authored by dialogue

Long-form human-AI interaction can produce artifacts neither side would have produced alone.

Test: What attribution models preserve human authorship and AI assistance honestly?

Disciplines: authorship studies, AI policy, IP research.

### 33. AI can act as an organizing prosthetic

The assistant can help preserve, sort, and extend patterns the user recognizes but cannot yet structure.

Test: Does AI-assisted organization increase follow-through without erasing user agency?

Disciplines: HCI, cognitive offloading, accessibility.

### 34. The assistant must not become the authority over the user's signal

A reflective AI should help organize, not own, the user's meaning.

Test: Does user-confirmation language reduce perceived coercion?

Disciplines: AI ethics, UX, psychology.

### 35. Emotional intensity should be treated as energy plus risk

Intensity can motivate action or distort judgment.

Test: Can an interface separate energizing force from overcertainty?

Disciplines: affective computing, AI safety, design research.

### 36. The same signal may require different modes

A body-symbol entry may need canonical, reflective, mythopoetic, or neutral handling depending on context.

Test: Do mode toggles improve user satisfaction and reduce overreach?

Disciplines: HCI, personalization, AI safety.

### 37. Exportable traces change the interaction

When users can export a session, the output becomes reviewable rather than ephemeral.

Test: Does exportability increase accountability and reduce hallucinated continuity?

Disciplines: recordkeeping, HAI, personal informatics.

### 38. Delayed meaning may be more accurate than immediate meaning

Some signals need time before interpretation stabilizes.

Test: Does revisiting entries after 7 or 30 days improve map accuracy?

Disciplines: longitudinal psychology, journaling research, HCI.

### 39. The system needs a "do not interpret yet" state

Sometimes the safest response is observation, not meaning.

Test: Does a wait-state improve safety in high-ambiguity inputs?

Disciplines: AI safety, clinical-adjacent UX, decision science.

### 40. The most interesting product behavior is uncertainty management

The core research value may be how the system handles not knowing.

Test: Can users maintain curiosity without false certainty when uncertainty is visibly structured?

Disciplines: epistemology, AI safety, HAI.

## Researcher invitation

A researcher reviewing this repo can treat Mirror Cartographer as a testbed for:

- reflective AI UX
- uncertainty labeling
- symbolic interface design
- embodied input systems
- typo/nonstandard-language signal weighting
- user correction loops
- epistemic safety in emotionally salient AI conversations
- long-context continuity and authorship
- research-to-product translation
- public-proof artifact design
