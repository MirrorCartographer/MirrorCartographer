# Public Quality Check

Use this checklist before publishing a public file.

## A file should include

- clear purpose
- specific audience
- visible example
- useful distinction
- next action
- boundary note

## A file should avoid

- generic language
- vague claims
- unsupported certainty
- empty categories
- private source material
- unclear invitation

## Strong public artifact test

A useful artifact helps a reader understand:

- what this is
- why it matters
- how to enter
- what can be tested
- what remains unfinished

## Rewrite rule

If the file could belong to any project, make it more specific.
