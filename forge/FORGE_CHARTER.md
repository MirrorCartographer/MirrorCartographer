# The Forge

The Forge is Mirror Cartographer's outside-ChatGPT build loop.

Its purpose is simple:

**Every serious interaction should leave behind a stronger artifact.**

The Forge is not a person and not a promise of unlimited autonomous work. It is a repository-native system for turning prompts, issues, fragments, and demonstration ideas into structured files that can be reviewed, improved, and published.

## What the Forge does

The Forge converts a fragment into one or more public-safe artifacts:

- demonstration card
- research map
- continuity packet skeleton
- evidence boundary map
- symbol glossary entry
- animal-care timeline template
- public webpage section
- GitHub issue plan
- next-action checklist

## Inputs

A Forge input should contain:

- title
- fragment or problem
- why it matters
- desired output
- public/private sensitivity
- source notes
- next action

## Outputs

A Forge output should contain:

- question
- why it matters
- input
- transformation
- proof of value
- boundary
- next build

## Operating rule

If a tool fails, the failure becomes a requirement.

If a result is boring, it is incomplete.

If a claim is unclear, label it.

If a file is public-facing, protect private source material by default.

## First outside-ChatGPT ability

The first ability is a GitHub Actions workflow that can generate a new public demonstration file from issue-style text or manual workflow input.

This does not require ChatGPT to remain open. It uses repository code to produce structured artifacts inside the repo.
