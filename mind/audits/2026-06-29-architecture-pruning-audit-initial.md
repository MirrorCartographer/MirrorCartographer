# Architecture Pruning Audit — Initial

Date: 2026-06-29  
Status: first lifecycle classification scaffold  
Purpose: distinguish active architecture from experiment, archive, superseded material, and retired claims.

## Lifecycle states

- Proposed: described but not yet tested.
- Experimental: testable and under evaluation.
- Active: currently recommended for use.
- Superseded: replaced by a stronger or narrower artifact.
- Archived: historically useful but not current guidance.
- Retired: unsupported, redundant, or too vague.

## Audit result

The current architecture is not too large yet, but it is at risk of accumulating method layers faster than test results. Future additions should either:

1. execute an existing test;
2. normalize an existing claim;
3. downgrade an overclaim;
4. retire a redundant artifact.

New evidence maps should be created only when they add independent evidence or a new falsification path.
