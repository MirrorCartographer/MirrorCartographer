# Start Here — Human Guide

Mirror Cartographer is not just a symbolic project. It is becoming a claim-governance system for human-AI co-created knowledge.

The simplest question this repo asks is:

> What would make this claim more or less trustworthy?

## What to read first

1. `mind/README.md`  
   Shows the whole lane structure.

2. `mind/evidence-quality/source-reliability-matrix.md`  
   Explains how sources are weighed.

3. `mind/claims/`  
   Holds current claims and statuses.

4. `mind/ledgers/negative-result-ledger.md`  
   Shows failed, weakened, or inconclusive claims.

5. `mind/public-proof/`  
   Holds artifacts meant for outside readers.

## What this repo is not claiming

It is not claiming that MC is clinically validated.

It is not claiming that symbolic maps diagnose anything.

It is not claiming that citations alone prove truth.

It is not claiming that a GitHub commit is an audit trail.

It is not claiming that AI-generated evidence maps prove commercial value.

## What it is claiming

MC is building a structure for:

- separating fact from inference;
- tracking evidence quality;
- preserving uncertainty;
- downgrading weak claims;
- testing symbolic and reflective outputs;
- keeping AI co-created knowledge auditable.

## Current strongest area

Evidence governance.

## Current weakest area

Empirical validation.

Many claims are supported as design principles or governance boundaries. Few have been tested directly in MC.

## What would make MC stronger

Not more beautiful language.

Not more citations.

Not more evidence maps.

The next real improvement is executed audits and pilots:

- source-to-claim support audit;
- evidence reliability scoring;
- accessibility validation;
- body-map safety test;
- symbolic-overinterpretation test;
- independent reviewer comparison.

## Reader rule

When reading any claim, ask:

- What source supports it?
- Is the source direct or indirect?
- What would contradict it?
- Has MC itself tested it?
- What is still unknown?
