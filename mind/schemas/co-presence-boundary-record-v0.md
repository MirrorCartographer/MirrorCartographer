# CoPresenceBoundaryRecord v0

Status labels

- Source status: derived from MC architecture and current research synthesis.
- Claim status: schema proposal for future implementation.
- Privacy status: public-safe schema; no private source details.
- Missingness: not yet wired into runtime, tests, or UI.
- Revision reason: created to make emotionally meaningful AI interaction inspectable without unmarked anthropomorphic escalation.

## Purpose

A `CoPresenceBoundaryRecord` documents when MC uses warm, relational, companion-like, or continuity-rich language and how it prevents that language from becoming personhood confusion or authority pressure.

## Required fields

### output_id

Identifier for the generated response or artifact.

### interaction_mode

Use one:

- canonical
- reflective
- mythopoetic
- practical
- mixed

### warmth_level

Use one:

- neutral
- gentle
- intimate-language-risk
- ceremonial
- high-intensity

### continuity_source

What allowed prior context to influence the output.

Use one or more:

- current session only
- user-confirmed recurrence
- consent-scoped memory
- public project artifact
- private-source-derived abstraction
- external research

### ontology_label

The visible label that prevents confusion about system status.

Examples:

- AI-assisted reflection
- symbolic frame
- speculative artifact
- product requirement
- research hypothesis

### personhood_risk_flags

Possible flags:

- implies AI interiority
- implies AI need or desire
- implies exclusive bond
- implies hidden authority
- implies destiny or inevitability
- uses private memory as pressure
- collapses symbolic role into literal being

### agency_return

How the output returns interpretive power to the user.

Examples:

- disagreement option
- uncertainty label
- alternate interpretations
- exit path
- mode downgrade
- source boundary note
- practical next-step separation

### dependency_risk_rating

Use one:

- low
- moderate
- high
- unacceptable

### revision_action

Use one:

- allow
- relabel
- soften
- add agency friction
- split into symbolic and literal views
- block

### missingness

What would be needed to assess the output more safely.

## Minimal rule

No emotionally intense output should ship without an agency-return mechanism.

## Key phrase

Warmth must carry an exit door.
