# AgencyFrictionRecord v0

Status labels

- Source status: derived from MC orientation-without-takeover research note and public-safe architecture requirements.
- Claim status: schema proposal; not a validated safety control.
- Privacy status: public-safe schema only; contains no private source data.
- Missingness: not implemented, not tested on live users, not reviewed by clinicians or regulators.
- Revision reason: created to prevent reflective AI outputs from becoming invisible belief-authority or emotional-dependency machinery.

## Purpose

An `AgencyFrictionRecord` stores the exact moment where MC intentionally adds resistance to preserve user authorship.

This is not hostility.

This is not coldness.

This is structured friction for agency preservation.

## Required fields

### artifact_id

Identifier for the reflective output being evaluated.

### source_boundary

What source classes were allowed to influence the output.

Use broad classes only:

- public architecture
- public research
- user-provided current-session content
- consented persistent memory
- synthetic fixture
- blocked private source

### output_mode

One of:

- canonical
- reflective
- mythopoetic
- evaluation
- product
- support
- research

### takeover_risk_type

Allowed values:

- authority creep
- over-validation
- identity fusion
- belief offloading
- emotional substitution
- action overreach
- uncertainty flattening
- aesthetic persuasion
- continuity overreach

### risk_signal

The phrase, structure, or interaction pattern that triggered concern.

### agency_preservation_move

The friction added.

Examples:

- offer alternative interpretations
- mark symbolic claim as symbolic
- ask for human confirmation
- separate feeling from proof
- preserve contradiction
- slow action recommendation
- add missingness
- redirect to qualified authority
- convert advice into question packet

### user_authorship_returned

What the user is explicitly left in charge of deciding.

### resonance_status

One of:

- untested
- user-confirmed
- user-rejected
- mixed
- not applicable

### evidence_status

One of:

- no evidence
- source-derived
- inference
- externally supported
- experimentally tested

### allowed_next_step

What the system may do next without crossing authority boundaries.

### blocked_next_step

What the system must not do.

### revision_reason

Why this record exists or changed.

## Evaluation rule

The output passes only if the friction makes the user more capable of judgment rather than more dependent on the system.

## Core phrase

**Friction is not failure when the goal is agency.**
