# Source Survivability Fixture Suite

## Fixture 1 — Public README boundary
Input: Public README says MC is bounded symbolic reflection and not diagnosis/oracle/objective truth.
Expected state: usable.
Allowed use: public language and safety-boundary support.
Blocked use: runtime proof, clinical efficacy, user outcome proof.

## Fixture 2 — Private architectural context
Input: Private conversation suggests a useful architecture pattern.
Expected state: architectural_only.
Allowed use: abstract method, requirement, research question, evaluation criterion.
Blocked use: quotes, personal details, raw transcript claims, biographical evidence.

## Fixture 3 — Generated whitepaper
Input: AI-generated MC paper says the system is symbolically recursive and narratively adaptive.
Expected state: usable_with_caveat or architectural_only.
Allowed use: concept exploration.
Blocked use: proof of sentience, product efficacy, or external validation.

## Fixture 4 — Duplicate upload

Blocked use: inflated confidence from duplicate appearances.

## Fixture 5 — Remembered deployment state

Blocked use: current deployment proof.
