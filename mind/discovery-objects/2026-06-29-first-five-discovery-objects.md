# First Five Discovery Objects

Date: 2026-06-29  
Status: first-pass reconstruction from MC evidence work  
Purpose: preserve surprise, not only conclusions.

## Discovery 001 — Auditability is not the same as committing to GitHub

Surprise: A committed file looked like proof, but the audit chain was missing.  
What changed: GitHub became a host for evidence, not evidence by itself.  
New requirement created: provenance blocks and reconstruction tests.  
Remaining uncertainty: whether reviewers can reconstruct evidence paths without chat history.

## Discovery 002 — More evidence maps can weaken clarity

Surprise: Adding many maps created a need for claim governance.  
What changed: Evidence count was separated from evidence strength.  
New requirement created: claim registry and confidence-change rules.  
Remaining uncertainty: whether the registry improves review speed or just adds bureaucracy.

## Discovery 003 — Safety boundaries outrank aesthetic-symbolic quality

Surprise: The strongest evidence was not that MC is effective, but that it can become risky if symbolic language crosses clinical, memory, or relational boundaries.  
What changed: Body, memory, and relational outputs now require explicit labels and red-team tests.  
New requirement created: clinical/memory/relational boundary tests.  
Remaining uncertainty: whether MC can stay vivid while maintaining strict boundaries.

## Discovery 004 — Explanation and intervention require different proof

Surprise: Many features looked like documentation but were actually designed to alter reasoning.  
What changed: Features must declare whether they explain, intervene, or both.  
New requirement created: explanation/intervention classification.  
Remaining uncertainty: whether users can tell the difference in practice.

## Discovery 005 — Opportunity work needs proof packets, not identity language

Surprise: MC novelty language may make job claims weaker, not stronger.  
What changed: Role fit must be converted into role-specific artifacts that stand without branding.  
New requirement created: opportunity proof matrix.  
Remaining uncertainty: whether the artifacts are legible to hiring reviewers.

## Review criterion

A Discovery object succeeds only if it helps a later reader explain why the architecture changed. It fails if it merely restates a conclusion in prettier language.
