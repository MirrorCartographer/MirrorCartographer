# Governance Artifact Manifest Helper Implementation

Date: 2026-07-03
Status: executable helper plus minimal fixture descriptors
Public safety: public-safe; no private user material; no personal identifiers; no raw session content.

## Architecture question

How should MC define the first executable `tools/lib/governance-artifact-manifest.mjs` implementation and minimal helper fixtures so path normalization, byte digests, public-safety blocks, and schema validation are proven before canonical JSON replay and ADR replay depend on it?

## Current-source research basis

- Node's file-system API supplies the stable local primitives MC needs for manifest generation: directory creation, file reads, stat byte sizes, temporary write, and rename finalization.
- Node's `crypto.createHash()` is the correct dependency-free SHA-256 primitive for file-byte digests.
- SLSA provenance separates artifact subjects, digests, materials, builder identity, and build metadata. MC should borrow this separation without requiring full provenance attestation inside the helper.
- GitHub artifact attestation and Actions documentation preserve the same boundary: CI may upload, retain, and attest artifacts, but artifact meaning should be generated by the producing tool.
- JSON Schema 2020-12 remains the right typed boundary for manifests, but the schema should stay shallow and fixed-shape because recent JSON Schema research shows modern schema features can create avoidable complexity.

## Updated understanding

The manifest helper should now be treated as a small executable custody kernel. Its job is not to decide whether a canonical JSON fixture, ADR index, or governance graph is correct. Its job is to make any governance replay output directory self-describing and verifiable.

The implementation boundary is therefore:

1. Domain replay tools write their own domain files.
2. Domain replay tools pass declared file roles to the helper.
3. The helper validates relative paths, reads actual bytes from disk, computes SHA-256, scans text-like files for public-safety blocks, builds a stable manifest envelope, validates it shallowly, and writes canonical `manifest.json`.
4. GitHub Actions uploads the artifact directory and consumes the generated summary without recreating path constants.

This converts `manifest.json` from a loose CI file listing into a deterministic artifact-custody record.

## Implemented artifact set

### `tools/lib/governance-artifact-manifest.mjs`

Added a dependency-free ESM helper using only Node built-ins plus the existing canonical JSON helper. It exposes:

- `GovernanceArtifactManifestError`
- `normalizeArtifactPath(relativePath)`
- `resolveArtifactRoot(outDir)`
- `hashFileSha256(filePath)`
- `describeArtifactFile({ root, path, role, mediaType })`
- `scanArtifactPublicSafety({ root, files, policy })`
- `buildArtifactManifest({ artifactKind, producer, root, files, summary, exitBehavior, publicSafety })`
- `validateArtifactManifest({ manifest })`
- `writeArtifactManifest({ root, manifest })`
- `createArtifactManifestDirectory({ root, files, manifestInput })`

### `mind/schemas/governance.replay.artifact.manifest.v1.schema.json`

Added a shallow JSON Schema boundary for the helper output. It requires:

- `schema`
- `artifactKind`
- `producer`
- `summary`
- `exitBehavior`
- `publicSafety`
- `files`

It rejects absolute paths, home paths, drive-letter paths, path traversal, backslashes, and unknown top-level fields.

### `mind/fixtures/governance.replay.artifact.manifest-helper.v1/pass-minimal-file-set.json`

Added the first positive helper fixture descriptor. It defines a minimal generated artifact directory containing:

- `result.json` as `replay-result`
- `summary.md` as `summary`

The expected manifest is public-safe, deterministic, and contains two sorted file records.

### `mind/fixtures/governance.replay.artifact.manifest-helper.v1/fail-invalid-path.json`

Added the first negative helper fixture descriptor. It proves the helper must reject `../escape.json` before hashing or manifest write.

## Design constraints now locked

- Manifest paths are root-relative, slash-separated, and never absolute.
- The helper hashes bytes from disk, not caller-provided digest fields.
- The helper sorts file records by path.
- The helper writes canonical JSON through the shared canonical JSON kernel.
- The manifest contains no wall-clock timestamp, hostname, username, branch, run URL, or local absolute path.
- Public-safety blocks return stable check codes without copying offending content into the manifest.
- Domain-specific check schemas remain outside the helper.

## Acceptance checks for the next implementation pass

- Running the helper twice over the same files produces byte-identical `manifest.json`.
- `pass-minimal-file-set.json` can be replayed by a small local runner.
- `fail-invalid-path.json` fails with `GOVERNANCE_ARTIFACT_PATH_INVALID` and writes no manifest.
- The canonical JSON replay tool delegates manifest generation to the helper.
- The ADR index replay tool can later delegate to the same helper without coupling to canonical JSON checks.
