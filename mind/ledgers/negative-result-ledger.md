# Negative Result Ledger

Purpose: preserve failed, inconclusive, or weakened claims so the GitHub mind does not become a confirmation machine.

Status meanings:

- Failed: test contradicted the claim or showed the feature does not work as stated.
- Inconclusive: test did not support or refute the claim.
- Abandoned: claim is not measurable enough to continue.
- Downgraded: claim wording was too strong and has been narrowed.

## Ledger rule

Every failed or inconclusive test must record:

- claim affected;
- expected outcome;
- observed outcome;
- likely reason;
- whether confidence increased, decreased, or stayed unchanged;
- next action or retirement decision.
