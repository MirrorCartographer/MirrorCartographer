# Source Boundary Bill of Materials

Date: 2026-06-29
Status: Public-safe research note
Attractor: Source Boundary

## Core finding

Mirror Cartographer needs a **Source Boundary Bill of Materials**: a structured record of what kinds of context shaped a public artifact, what kinds were excluded, what transformations occurred, and what claims remain allowed after redaction.

This is not a list of private sources. It is a boundary ledger for public-safe cognition.

## Public-safe thesis

A public artifact can be privacy-safe and still be epistemically unsafe if the reader cannot tell:

- whether it came from public documents, private context, external research, model inference, or synthetic examples;
- whether private context was used as evidence or only as architectural orientation;
- what claim modes are permitted;
- what was removed, generalized, compressed, or withheld;
- what missingness still matters.

MC should therefore publish not raw provenance but **source-boundary provenance**.

## Claim status

- Supported claim: MC already requires uncertainty labels, mode boundaries, contradiction logs, trajectory nodes, and public compression before release.
- Supported claim: Current AI memory/governance research increasingly treats memory as provenance-bearing infrastructure, not just stored personalization.
- Design inference: MC can turn this into a public-safe artifact-release primitive: every public output should carry a source-boundary bill.
- Speculative claim: Source-boundary provenance may become more useful than raw provenance for public artifacts because it proves the crossing without exposing protected material.

## Revision reason

Previous MC runs established public-safe compilation, influence graphs, redaction fidelity, symbolic translucence, and release readiness. This note adds the missing material-accounting layer: a bill of materials for the source-boundary crossing itself.

## Implementation direction

Each public artifact should include a compact SourceBoundaryBOM with:

1. artifact id and release state;
2. source class inventory;
3. private-use class, if any;
4. transformation chain;
5. removed categories;
6. retained structural signal;
7. claim mode permissions;
8. missingness statement;
9. revision reason;
10. release decision.

## Key phrase

**Do not expose the ingredients. Expose the boundary conditions of the recipe.**
