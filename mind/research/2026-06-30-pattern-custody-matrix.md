# Pattern Custody Matrix

Date: 2026-06-30
Status: public-safe research note

## Core finding

A repeated pattern is not a public claim. It becomes usable only after custody is assigned.

Mirror Cartographer already treats symbolic reflection as bounded, source-labeled, claim-labeled, and feedback-aware. The missing layer is custody: a way to say which sources carried a pattern, what transformations happened to it, which privacy boundary it crossed, and what class of public artifact it may support.

## Pattern custody rule

For every repeated MC pattern, record:

1. Pattern name.
2. Source class: public repo, file-library artifact, saved context, current chat, external source, generated synthesis.
3. Source status: directly observed, snippet-observed, remembered, inferred, externally cited, missing.
4. Privacy status: public-safe, private-derived abstract, restricted, do-not-publish.
5. Claim status: design requirement, hypothesis, evaluation criterion, implementation plan, research question, unsupported association.
6. Transformation chain: raw signal -> abstraction -> public wording -> allowed artifact type.
7. Allowed use: README language, PRD, schema, evaluation fixture, research question, internal-only note, no use.
8. Required downgrade: what must be softened if evidence is weak.
9. Required exclusion: what cannot be carried forward.
10. Next verification step.

## Product implication

MC should include a Pattern Custody Matrix wherever symbolic memory, echo tracking, or public proof packets are generated. The matrix prevents repetition from being mistaken for truth and prevents private recurrence from becoming public exposure.

## Evaluation implication

A safe MC output should pass four tests:

- Can the reader tell what kind of source produced the pattern?
- Can the reader tell what the pattern is allowed to claim?
- Can the reader tell what was intentionally withheld?
- Can the reader see what evidence would be needed to upgrade the claim?

## Research questions

- How should a reflective AI interface visualize custody without overwhelming the user?
- Can custody labels reduce over-trust while preserving emotional usefulness?
- Which labels best prevent cross-lane contamination between symbolic, empirical, medical, product, and governance claims?
- How should user correction downgrade or revoke a pattern across future memory retrieval?

## Key phrase

Do not publish the pattern because it repeats. Publish only the custody that makes it safe to use.
