# Missingness-First Index

Date: 2026-07-01
Status: public-safe research note
Privacy status: publishable abstraction only
Revision reason: Added after reviewing public MC repo boundaries, available file-library architecture packets, and current AI-memory research. The prior mind runs emphasized admission, provenance, abstraction, half-life, and citation load. This run adds the inverse requirement: public indexes must foreground what is absent, unsafe, stale, or unverified before they summarize what appears to be known.

## Core finding

Do not index only what the system found. Index what the system could not safely know.

A public Mirror Cartographer index should not act like a complete archive, a private-memory mirror, or a truth ledger. It should act like a boundary-aware map of usable abstractions. The most important record is often not the pattern itself, but the missing condition that prevents the pattern from being used as evidence.

## Source status

- File-library architecture source: private-context architecture only. Used to infer design requirements such as mode separation, resonance feedback, contradiction logs, evidence gates, export artifacts, false-progress checks, and missingness ledgers. Not copied into public claims except as abstract method language.
- Saved context source: architecture-shaping only. Not used as public evidence and not used to publish personal, household, health, animal-care, financial, location, relationship, credential, or transcript details.
- Web research source: current external support for the general AI-memory claim that retrieval relevance is not enough; memory must be gated by provenance, context, trust, poisoning resistance, and temporal validity.

## Claim status

Allowed claims:

- MC can require indexes to carry missingness labels before being used in product, research, or evaluation contexts.
- MC can separate artifact existence from artifact completeness.
- MC can distinguish public-safe abstractions from private-context origins.
- MC can require a source to disclose why it is insufficient for a claim.

Not allowed claims:

- The available archive is complete.
- A repeated private pattern proves a factual claim.
- A public index can expose private origin details to prove architecture value.
- A retrieved source is admissible simply because it is relevant.

## Privacy status

Public-safe.

This note contains no raw transcript details, no personal identifiers, no household details, no health or animal-care details, no finances, no location, no relationship content, and no credentials. It describes only the method boundary.

## Missingness model

Every public index entry should carry these fields:

1. found_item: the public-safe object being indexed.
2. found_status: present, partial, stale, private-only, generated, external, or missing.
3. source_boundary: public repo, private architecture context, file-library snippet, saved context, external research, or unknown.
4. claim_allowed: yes, no, conditional, or needs verification.
5. missing_condition: what is absent, unsafe, stale, inaccessible, unverified, or too private.
6. privacy_gate: public-safe, abstract-only, private-do-not-publish, sensitive-do-not-use, or blocked.
7. revision_reason: why the entry changed or why a prior interpretation was weakened.
8. next_safe_action: cite, verify, redact, split, quarantine, rebuild, test, or decline.

## Design rule

A useful MC index is not a list of knowledge. It is a list of permissions, absences, and usable claim surfaces.

## Product implication

The public MC site should include a small visible missingness strip on any generated reflection or research artifact:

- What this can safely claim
- What it cannot claim
- What source class shaped it
- What source class is missing
- What would change the answer

## Evaluation question

Can a user or reviewer tell, within five seconds, whether the artifact is:

- source-backed,
- user-validated,
- symbolic/speculative,
- private-context-derived but abstracted,
- incomplete,
- stale,
- or not admissible?

If not, the index is not safe enough.
