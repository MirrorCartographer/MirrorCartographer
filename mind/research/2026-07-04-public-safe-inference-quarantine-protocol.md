# Public-Safe Inference Quarantine Protocol

## Core finding

Mirror Cartographer needs a **Public-Safe Inference Quarantine Protocol**: a publication guard that separates safe abstracted methods from unsafe inferred residue.

A public artifact can avoid direct private details and still leak private-context shape through derived categories, ordering, examples, priorities, metaphors, topology, or unusually specific implementation choices. The system therefore needs a quarantine step for inferences that are not themselves raw private facts, but were meaningfully shaped by private source material.

## Operating line

> A public-safe artifact should not only remove private facts; it should quarantine any inference whose specificity, ordering, metaphor, or implementation pressure would let private context echo through the abstraction.

## Source status

- **Available source classes used:** prior Mirror Cartographer conversation context, saved architectural memory, and visible GitHub commit history.
- **Repository state observed:** recent public-safe research commits already cover claim gates, maturity ladders, traceability, source rehydration, assumption expiry, interface contracts, and fixture boundaries.
- **Private source handling:** private-context material was used only to understand architectural pressure and recurring failure modes.
- **Public source handling:** no raw transcript, household, health, animal-care, financial, location, relationship, credential, or identity-specific detail is included.

## Claim status

- **Claim type:** product-governance requirement and evaluation protocol.
- **Claim strength:** design-level finding, not empirical proof.
- **Evidence class:** recurring architecture pattern inferred from the existing MC privacy-safety research sequence and the stated publication boundary.
- **Allowed public claim:** MC should treat derivative inference as a privacy surface, not only explicit source text.
- **Disallowed public claim:** any claim that a specific private event, person, symptom, animal, relationship, place, financial condition, or transcript produced this requirement.

## Privacy status

- **Privacy classification:** public-safe abstraction.
- **Redaction level:** no private examples; no biographical anchoring; no raw transcript fragments; no private scenario fixtures.
- **Risk class:** indirect leakage by shape, not direct leakage by content.
- **Required boundary:** do not publish examples whose variable structure can be reverse-mapped to private context.

## Missingness

The current public-safe research stack still needs explicit tests for:

1. **Inference residue:** Does the output preserve a private-derived topology even after labels are sanitized?
2. **Priority leakage:** Does the order of backlog items reveal private urgency or lived context?
3. **Metaphor leakage:** Does a symbol, image, or repeated phrase carry private-context traceability even without names?
4. **Fixture leakage:** Does a synthetic example reproduce the relational, temporal, or embodied shape of a private source?
5. **Implementation pressure leakage:** Does a product requirement reveal what private source type forced its existence?

## Protocol

Before a finding is added to a public artifact, evaluate it through five quarantine gates.

### 1. Specificity gate

Ask whether the finding is more specific than the public evidence warrants.

- **Pass:** the finding is expressed as a general method, requirement, criterion, or research question.
- **Quarantine:** the finding includes unusually narrow variables, sequences, sensory details, household-like structure, medical-like structure, animal-care-like structure, financial-like structure, location-like structure, relationship-like structure, or credential-like structure.
- **Release condition:** generalize variables until the method remains useful without preserving private topology.

### 2. Ordering gate

Ask whether the order of listed items encodes private priority.

- **Pass:** order is alphabetical, dependency-based, maturity-based, or explicitly arbitrary.
- **Quarantine:** order reflects private urgency, emotional salience, recency, crisis pressure, or unresolved personal stakes.
- **Release condition:** relabel the ordering basis or reorder by public-safe implementation dependency.

### 3. Metaphor gate

Ask whether the metaphor carries a private symbolic trace.

- **Pass:** metaphor is generic, domain-standard, or clearly product-level.
- **Quarantine:** metaphor is unusually personal, repeated across private context, or tied to a recognizable private symbolic system.
- **Release condition:** replace with neutral product, evaluation, or systems language.

### 4. Fixture-shape gate

Ask whether a synthetic test case preserves private context structure.

- **Pass:** fixture uses generic actors, neutral domains, invented timings, and public-safe variable distributions.
- **Quarantine:** fixture preserves relational count, event order, category mix, location pattern, medical/animal/financial structure, or transcript-derived phrasing.
- **Release condition:** rebuild the fixture from public domain-neutral constraints, not from private case transformation.

### 5. Necessity gate

Ask whether the public artifact needs this inference at all.

- **Pass:** inference directly improves safety, implementation, evaluation, documentation, or product clarity.
- **Quarantine:** inference is interesting but not needed for the public artifact.
- **Release condition:** demote to internal-only architecture note or remove.

## Evaluation criteria

A public-safe finding passes when:

- it can be explained without referring to private source classes;
- its examples can be generated independently from public product requirements;
- its priority is justified by implementation dependency, not private urgency;
- its language does not preserve a private symbolic signature;
- its usefulness survives after all private-shaped specificity is removed;
- its missingness is visible enough for future maintainers to avoid overclaiming.

## Public-safe implementation plan

1. Add an `inference_quarantine` field to public-safe research manifests.
2. Require each finding to declare whether it is:
   - `direct_public_source`,
   - `abstracted_private_context`,
   - `mixed_architecture_inference`, or
   - `quarantined_derivative`.
3. Add a `release_condition` for every quarantined derivative.
4. Add a `shape_leakage_review` checklist to fixture generation.
5. Add a regression test that compares public fixtures against banned private-shape patterns without storing the private patterns in the public repository.

## Public-safe index entry

- **Index label:** inference quarantine
- **Source boundary:** private context may inform architecture; it may not provide public examples, topology, priority order, or symbolic residue.
- **Claim boundary:** design requirement, not empirical validation.
- **Privacy boundary:** protects against derivative leakage after redaction.
- **Missingness boundary:** needs automated shape-leakage tests and manifest integration.

## Meaningful revision reason

This note extends the existing public-safe research stack by addressing a gap not fully covered by source rehydration, fixture boundaries, or redaction regression. Those gates focus on source material, examples, and future reconstruction. This protocol targets the middle layer: derivative inferences that are clean in wording but still private-shaped in structure.
