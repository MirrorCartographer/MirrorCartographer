## Core finding
Mirror Cartographer should allow meaning to detect cross-domain resonance while forcing evidence to remain local to the domain that can actually verify it.

This becomes the operating rule:

Meaning may connect lanes. Evidence must stay lane-specific.

## Product requirement
Every continuity packet must declare an `evidence_lane` before it can produce a public or professional-facing view.

Allowed starting lanes:
- product_design
- software_implementation
- user_research
- AI_governance
- education_or_literacy
- social_care_support
- medical_or_clinical_support
- creative_artifact
- business_or_income
- personal_reflection_private_only

## Required lane fields
Each lane must carry:
- lane name
- permissible evidence types
- prohibited claims
- review authority
- transformation rule
- privacy threshold
- missingness label
- next evidence-changing action

## Public-safe example
Input meaning pattern: "A recurring contradiction appears across several contexts."

Invalid output:
"The same cause explains every context."

Valid lane-split outputs:
- Product lane: "Add contradiction persistence to the state graph."
- AI-governance lane: "Test whether long-horizon agents show measurable objective drift before explicit policy failure."
- Social-care lane: "Convert the contradiction into a question packet for human review."
- Creative lane: "Use the contradiction as an aesthetic structure without calling it proof."

## Evaluation criteria
A packet passes the Evidence Lane Splitter if:
1. It names the domain being evaluated.
2. It lists evidence that would actually change the claim.
3. It blocks symbolic resonance from becoming causal assertion.
4. It keeps private material out of public views.
5. It preserves enough lineage to show what changed.
6. It defines the next test.
