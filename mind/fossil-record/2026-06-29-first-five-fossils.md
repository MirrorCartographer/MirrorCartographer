# First Five Fossil Records

Date: 2026-06-29  
Purpose: preserve failed or weakened ideas so they can teach instead of disappearing.

## Fossil 001 — GitHub commit as proof

Original idea: A GitHub update proves value.  
Why it failed: A commit proves a file was written, not that the claim is true or auditable.  
What survived: GitHub is useful as a public evidence host when provenance is explicit.  
Do not repeat: Treating version control as validation.

## Fossil 002 — MC as uniquely solving broad AI danger

Original idea: MC is uniquely able to solve major AI problems.  
Why it failed: The claim is too broad, not falsifiable, and unsupported by direct evidence.  
What survived: MC can create useful claim-boundary, provenance, and reflection-governance artifacts.  
Do not repeat: “Only MC can solve X” unless X is narrow, tested, and compared.

## Fossil 003 — More features equals better architecture

Original idea: Adding Discovery, Fossil, Attribution, Scope, Agency, and Ledger features automatically improves MC.  
Why it failed: Unvalidated features can add conceptual debt.  
What survived: Features are useful only when tied to decision quality, safety, or evidence clarity.  
Do not repeat: Adding components without retirement criteria.

## Fossil 004 — Symbolic vividness equals deeper truth

Original idea: Vivid symbolic reconstruction is inherently meaningful and safe.  
Why it failed: Vivid reconstruction can distort memory, inflate confidence, or create dependency.  
What survived: Symbolic work remains valuable when labels separate reported, inferred, symbolic, unknown, and contraindicated material.  
Do not repeat: Treating aesthetic resonance as evidence.

## Fossil 005 — Role fit equals application readiness

Original idea: If a role matches MC thinking, it is actionable.  
Why it failed: Hiring requires proof-of-skill, not conceptual resonance.  
What survived: MC artifacts can become work samples if packaged around role requirements.  
Do not repeat: Applying before proof packet exists unless the goal is exploration rather than readiness.

## Fossil test

Before reintroducing any fossilized idea, answer:

1. What failed last time?
2. What evidence changed?
3. What narrower claim is now being made?
4. What would falsify it this time?
