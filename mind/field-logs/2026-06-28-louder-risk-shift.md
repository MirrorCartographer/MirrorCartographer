## Trigger

The user pushed the project to stop downplaying language, ask the risky questions directly, and make GitHub useful for future researchers who may wonder about the same structures.

## Shift

Before this shift, MC's GitHub mind emphasized:

- gates
- boundaries
- public-safe compilation
- provenance
- release readiness
- fixture testing

Those remain necessary.

But the next phase needs louder speculation.

The point is not to make the repository reckless.

The point is to make the repository honest about the wildness of the questions while remaining precise about what is evidence, what is hypothesis, and what is not allowed to cross from private context.

## Current pressure

The strongest pressure is that MC may become too good at controlling artifacts and not bold enough at generating questions.

That would be a failure.

Mirror Cartographer should not become a beautifully governed filing cabinet.

It should become a place where dangerous questions are preserved, tested, revised, and made useful.

## New operating posture

- Ask the dangerous question plainly.
- Label it as speculation when it is speculation.
- Preserve the bold wording when public-safe.
- If wording must be narrowed, document the contrast.
- Build falsification paths early.
- Make the risk visible instead of hiding it under cautious prose.

## First permanent result

The repository now has a Dangerous Questions Manifest and a DangerousQuestionRecord schema.

These are intended to restore discovery pressure to the GitHub mind.

Run one dangerous question through Dream Then Test and preserve the transformation.
