# Choiceful Signal Interface

## Bridge

Animal-computer interaction, interoceptive AI, and reflective interface design point to the same architecture pattern: uncertain signals should be given a choiceful interface lane before they are interpreted, remembered, or acted on.

This bridge is not a claim that AI can diagnose bodies, translate animals, or infer inner truth. It is a design pattern for preserving weak signals while keeping interpretation reversible.

## Research basis

### Animal-computer interaction: give nonverbal users control without pretending certainty

Animal-computer interaction research adapts human-centered design methods for non-human users. DogPhone and related animal internet work explore interfaces where animals can initiate, ignore, or end interaction through familiar objects or species-appropriate touch patterns.

Useful concept for MC: the interface should not force every signal into language. It should allow initiation, refusal, pause, and ambiguity.

Design caution: observed engagement is evidence of interaction, not evidence that the designer correctly understands the animal's intention.

### Interoceptive AI: regulate before interpreting

The interoceptive machine framework treats internal-state monitoring, homeostatic regulation, allostatic anticipation, and enactive interaction as abstractions for adaptive AI architecture.

Useful concept for MC: before reflection becomes meaning or memory, the system should check the state of the signal: intensity, uncertainty, context, and action-risk.

Design caution: this is an architectural analogy, not a direct biological claim about the user or an animal.

### Multispecies coexistence: design for shared environments

Research on animals and autonomous mobility systems argues that autonomous systems increasingly operate in environments shared with animals and should account for physical impact, behavioral effects, accessibility, ethics, and disturbance.

Useful concept for MC: the field is shared. A reflection system that includes animals, body cues, environment, and AI outputs needs separate lanes so one type of signal does not overwrite another.

## MC concept

A Choiceful Signal Interface lets a user record a signal without forcing interpretation.

The interface separates:

1. Raw signal: what was noticed.
2. Source lane: human body, animal observation, environment, dream/symbol, external source, AI inference.
3. Choice lane: observe, pause, compare, ask for evidence, convert to creative symbol, make public-safe, or escalate to professional care when appropriate.
4. Confidence lane: known, likely, possible, unknown, or metaphor-only.
5. Effect lane: no outside action, private note, durable memory candidate, public-safe artifact, or external action request.
6. Reversal lane: delete, downgrade, anonymize, reclassify, or mark as unsupported.

## Product wedge

### Signal-without-meaning card

A small MC card asks:

- What did you notice?
- Where did it come from?
- What should MC do with it right now?
- How certain is the meaning?
- Should this stay private, become memory, become art, or remain unsorted?

The key wedge: MC becomes useful before it is certain.

## Visual metaphor spec

Use a small lantern held over water.

- Water = raw signal field.
- Lantern = attention without capture.
- Floating tags = source lanes.
- Shoreline = action boundary.
- Fog = uncertainty.
- Lock/seed icons = memory choice.

The card should feel like a safe holding container, not a form or diagnosis screen.

## Public-safe demo idea

Create five sample cards using harmless non-medical examples:

1. A dog keeps looking toward the door.
2. A user notices chest-tight metaphor in a song lyric.
3. A room feels easier after rearranging furniture.
4. A recurring color appears in journal entries.
5. A research paper changes the framing of a project.

For each card, show raw signal, lane, uncertainty, possible interpretations, and safe next move.

## Evaluation criterion

A non-technical user should be able to tell within 30 seconds:

- what was observed;
- what is fact versus inference;
- whether anything will be remembered;
- whether anything will affect the outside world;
- how to undo or downgrade the interpretation.

## Falsification checklist

This pattern fails if:

- the UI makes a medical, veterinary, psychological, or spiritual claim from a weak signal;
- AI inference is visually indistinguishable from observed fact;
- private material is accidentally converted into public artifact language;
- the user cannot find the undo/downgrade control;
- the system nudges escalation where observation would be safer;
- the system minimizes professional care when professional care is warranted.

## Next concrete experiment

Build a static prototype of five Signal-without-Meaning cards. Test whether a user can accurately classify source, certainty, privacy level, and next safe move without explanation.
