# Reflective Interface Screen Specs

Date: 2026-06-29
Status: product specification

## Screen 1: Pre-Entry Privacy Gate

Purpose: warn before disclosure, not after.

Fields shown before the user enters sensitive material:

- private: keep local or session-only
- safe to save: can become memory
- risky to share: contains other people, health, finances, legal, location, or identity details
- never store: allowed in session only

User action:

- choose save mode before entry
- choose whether third-party information is blocked or abstracted

## Screen 2: Draw Before Meaning

Purpose: slow interpretation down before language over-smooths it.

Interaction:

1. user makes a rough mark, symbol, body-location tap, color, or spatial placement
2. MC asks what is noticed, not what it means
3. AI offers multiple candidate meanings, all marked reflect-only
4. user accepts, rejects, revises, or leaves unresolved

Required fields:

- observed mark
- body location
- color/texture/temperature
- atmosphere
- candidate meaning
- uncertainty
- allowed influence

## Screen 3: Interpretation Card

Every AI interpretation must display:

- claim
- uncertainty
- evidence used
- memory used
- body/symbol signal used
- alternative explanations
- allowed influence: reflect-only / suggest-gently / decision-relevant / blocked
- user action: accept / revise / reject / quarantine

## Screen 4: Memory Receipt

Purpose: make memory visible and revocable.

Fields:

- what was stored
- what was inferred
- why it was saved
- what it is allowed to influence
- what it is blocked from influencing
- third-party exposure status
- expiry/review date
- edit/delete/quarantine buttons

## Screen 5: Co-Memory Ledger

Purpose: track authorship and AI influence.

Fields:

- user-origin
- AI-origin
- mixed-origin
- revised-origin
- contested-origin
- human wording retained
- AI wording accepted
- AI wording rejected
- user correction

## Screen 6: Semantic Commit Merge

Purpose: prevent new interpretations from silently overwriting old ones.

Fields:

- new memory
- old memory conflict
- what would be overwritten
- what would remain unchanged
- merge option
- keep separate option
- block influence option

## Screen 7: Organism Byline

Purpose: make multi-agent contribution visible.

Every artifact shows:

- proposed by
- challenged by
- revised by
- evidence checked by
- killed or blocked by
- final allowed influence

Example:

Proposed by Child. Mapped by Cartographer. Audited by Critic. Built by Engineer. Stored by Archivist.

## Screen 8: Adaptive Room State

Purpose: the interface changes based on cognitive mode.

States:

- sensing: body map, colors, rough marks, low language pressure
- interpreting: candidate meanings, uncertainty, contradictions
- testing: evidence map, alternative hypotheses, confidence
- deciding: influence scope, risk, next concrete move
- archiving: memory receipt, provenance, expiry

## Screen 9: Privacy Path Map

Purpose: show where data moved.

Fields:

- what entered
- where it moved
- which tool saw it
- what wrote memory
- what can reuse it
- what is blocked

## Screen 10: Printable Memory Object

Title: What This Memory Is Allowed to Become

Printable fields:

- source
- symbol
- meaning candidate
- confidence
- allowed influence
- forbidden use
- revision note
- date

Use cases:

- ritual object
- therapy-adjacent discussion aid
- personal archive
- accountability artifact
- public demo prop
