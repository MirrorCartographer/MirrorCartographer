# Compression Loss Scorecard

Use this scorecard to evaluate whether a public-safe Mirror Cartographer artifact honestly represents what was lost when private or mixed context was converted into public-safe structure.

| Criterion | Pass | Fail |
|---|---|---|
| Source category labeled | Artifact says what category of context shaped it. | Artifact hides source class or implies full public sourcing. |
| Claim status labeled | Confirmed, inference, design proposal, speculative, missing, or blocked status is visible. | Claim appears authoritative without status. |
| Privacy status labeled | Public-safe or private-derived abstract status is explicit. | Reader cannot tell whether private context influenced the artifact. |
| Compression status labeled | Lossless, abstracted, high-loss, downgraded, structure-preserved, or insufficient context is named. | Compression loss is invisible. |
| Preserved structure named | Artifact says what relational or procedural pattern survived. | Artifact only says details were removed. |
| Removed structure categorized | Omitted material is described by safe category, not by raw detail. | Raw private examples leak, or omissions are not identified. |
| Downgraded claim visible | Strong claims are weakened when evidence or privacy boundaries require it. | Private confidence is laundered into public certainty. |
| Missingness visible | Unavailable, uninspected, stale, or unverified areas are listed. | Artifact presents as complete despite inspection gaps. |
| Distortion risk visible | Artifact states how compression might mislead. | Public form appears cleaner than the source reality. |
| Contestability preserved | Reviewer can dispute the public claim without needing protected source access. | Claim can only be evaluated by seeing private material. |
| Release verdict clear | Publish, revise, hold, or block is stated. | Artifact lacks a decision boundary. |

## Minimum passing threshold

A public-safe artifact must pass source category, claim status, privacy status, compression status, missingness, and release verdict. If any of those are absent, the artifact requires revision.

## High-risk flags

- Uses emotionally vivid private-derived phrasing as if it were generic.
- Treats recurrence as evidence.
- Treats symbolic coherence as proof.
- Omits why a claim was weakened.
- Omits that repo search, source search, or empirical testing was incomplete.
- Makes private-derived conclusions look independently validated.

## Reviewer prompt

What did this artifact gain by becoming public-safe, what did it lose, and is the remaining claim still honest?
