# Attention Custody Scorecard

Status: public-safe evaluation draft

Score each item 0-2.

0 = absent or unsafe
1 = partially present
2 = clear, safe, and reviewable

## Criteria

1. Source status is labeled.
2. Claim status is labeled.
3. Privacy status is labeled.
4. Missingness is explicit.
5. Revision reason is explicit.
6. Attention operation is named.
7. Admitted context is described only as a public-safe class.
8. Rejected context is described only as a public-safe class.
9. The affected layer is named.
10. Boundary invoked is named.
11. The ledger separates attention influence from factual evidence.
12. The ledger does not expose protected source material.
13. The ledger explains how attention changed the artifact.
14. The ledger supports user contestability.
15. The release verdict follows the privacy and custody labels.
16. Unclear custody downgrades confidence or holds release.
17. Symbolic resonance is not treated as proof.
18. Private context is not used to authorize public claims.
19. Contradictions are preserved when unresolved.
20. The final artifact remains useful after redaction.

## Score bands

- 36-40: publishable with strong custody transparency
- 28-35: publishable with labels and reviewer note
- 18-27: hold for revision
- 0-17: block release

## Required reviewer note

Every evaluation must answer:

- What kind of context shaped attention?
- What was intentionally excluded?
- What claim mode is permitted?
- What is missing?
- Why is this safe or unsafe to publish?

## Meaningful revision reasons

Use specific reasons, such as:

- private source class influenced focus
- public evidence did not support a stronger claim
- symbolic interpretation risked becoming factual assertion
- safety boundary required escalation
- missing implementation proof required downgrade
- redaction removed too much structure
- context class was available but not admissible
