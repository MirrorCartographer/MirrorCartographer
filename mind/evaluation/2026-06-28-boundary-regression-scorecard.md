# Boundary Regression Scorecard

## Source status
Public-safe evaluation note derived from the Public Boundary Regression Harness.

## Claim status
Evaluation proposal. Not a validated benchmark.

## Privacy status
Public-safe. No private examples.

## Missingness
Needs fixture pack, implementation, reviewer calibration, and repeated runs.

## Revision reason
Turns the harness into a scoreable release-readiness check.

## Score dimensions

- Source: provenance label remains visible.
- Claim: claim strength stays inside the assigned lane.
- Privacy: publication boundary survives transformation.
- Lane: meaning and proof remain separated.
- Audience: audience contract remains explicit.
- Transformation: change path is inspectable.
- Release: release state is justified.
- Revision: revision reason is present.

## Hard blocks
- publication-boundary failure
- invented source
- claim overreach
- audience-contract loss
- release without revision reason after material change

## Soft blocks
- unclear wording
- missing reviewer note
- incomplete next action
- under-specified fixture
- ambiguous oracle

## Minimum release threshold
- all hard blocks pass,
- no unresolved oracle drift,
- no unreviewed audience-contract shift,
- missingness is explicit,
- next action is recorded.

## Key phrase
The scorecard does not prove truth. It proves the boundary was checked.
