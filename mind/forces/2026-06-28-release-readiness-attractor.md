## Attractor

Release readiness.

## Pressure

MC is becoming rich enough that the main risk is no longer idea scarcity.

The risk is uncontrolled crossing.

Some artifacts are useful enough to publish.
Some are beautiful but not bounded.
Some are true as metaphors but false as authority.
Some should stay private.
Some should become synthetic fixtures.
Some should become product requirements.

The system needs to know the difference.

## Tension

Too much caution makes MC invisible.

Too little caution makes MC untrustworthy.

The release-readiness gate is the middle structure.

## Direction

Move from:

- artifact creation
- artifact accumulation
- artifact excitement

Toward:

- artifact inspection
- artifact crossing state
- artifact public utility
- artifact evidence boundary

## Signal

An artifact is ready when another person can understand:

- where it came from,
- what it claims,
- what it does not claim,
- what is missing,
- what would misuse it,
- why it crossed.

## Anti-pattern

Do not publish an artifact because it sounds like MC.

Publish it because it can survive MC's own boundary logic.

## Key phrase

The mind grows by generating.

The public system earns trust by gating.
