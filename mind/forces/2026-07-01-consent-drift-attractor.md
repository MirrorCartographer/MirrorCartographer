# Attractor: Consent Drift

## Name

Consent Drift

## Core finding

**Permission is not preserved by memory. It has to be re-checked at the boundary where memory becomes action.**

## Why it matters

Mirror Cartographer depends on continuity. Continuity is powerful because it lets patterns survive across time. That same survival creates risk: a pattern remembered for private reflection may later be reused for public synthesis, evaluation, product strategy, or automation.

Consent Drift names the moment when a source remains available but its permission state no longer matches the use being attempted.

- Source Survivability asks whether a source can carry a claim.
- Pattern Custody asks whether a pattern may testify.

- Revision Reason Ledger records why a change was made.

- Context Custody Chain allows context to shape the map without becoming the map.

- Abstraction Survivability tests whether a rule survives without private source details.

Consent Drift adds: **even after abstraction succeeds, the permission state must be checked again when use changes.**

## Public-safe formulation

A private source can teach a public rule. The public rule must not preserve the private source. The permission to use that rule must still be reviewed when the artifact moves domains.

## Failure pattern

1. Source admitted for one purpose.
2. Pattern extracted.
3. Pattern becomes useful.
4. Use boundary changes.
5. System treats prior admission as current permission.
6. Private material, authority, or inference leaks through a public artifact.

## Corrective pattern

1. Label source status.
2. Label claim status.
3. Label privacy status.
4. Label consent state.
5. Run public-survival test.
6. Record missingness.
7. Record revision reason.
8. Re-check consent at export, publication, sharing, and memory-write boundaries.

## Operational phrase

**A remembered source is not an authorized source.**
