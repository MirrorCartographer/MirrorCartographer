## Attractor

Symbolic Translucence

## Pull

MC wants to become a system where symbolic interpretation is powerful without becoming mysterious authority.

The pull is toward visible scaffolds:

- claim labels
- lens labels
- missingness labels
- source-boundary labels
- agency controls
- resonance/proof separation

## Counter-pull

The opposite attractor is magical opacity.

Magical opacity feels impressive because the system speaks fluently, beautifully, and personally.

But if the user cannot see how the map was made, the map can become too persuasive.

## Product implication

Every meaningful MC artifact should expose a small public-safe legend.

The legend does not need to reveal hidden internals.

It needs to reveal enough for contestability.

## Evaluation implication

MC should be scored not only on quality of output, but on whether the user can inspect and challenge the interpretive move.

## Release implication

An artifact is weaker if it is beautiful but not inspectable.

An artifact is stronger when beauty carries a visible scaffold.

## Key phrase

Do not remove the magic. Put windows in it.
