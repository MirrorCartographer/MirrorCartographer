# Attractor: Contestability Receipt

## Core attractor

A claim that cannot be contested is not bounded; it is only decorated.

## Source status

Public-safe synthesis from MC repository boundaries, public-safe implementation materials, and current AI-memory trust-boundary research.

## Claim status

Conceptual attractor and product-governance proposal.

## Privacy status

Public-safe. No private details.

## Meaning

MC's public-safe architecture should not end at redaction or release. It should preserve a living correction surface around claims, especially claims shaped by private context, symbolic interpretation, long-term memory, or mixed-source synthesis.

## Design pressure

If a claim can persist, it must be contestable.

If a claim can become public, it must be correctable without exposing the private source that shaped it.

If a symbolic interpretation becomes useful, it must still remain downgradeable.

## Boundary sentence

Do not make the private source public. Make the public claim accountable.

## Missingness

Implementation was not verified in the live UI or codebase during this run.

## Revision reason

Added because previous public-safe gates focus on what may cross into public; this attractor focuses on how public claims remain reviewable after crossing.
