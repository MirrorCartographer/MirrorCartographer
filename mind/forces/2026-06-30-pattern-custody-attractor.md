# Pattern Custody Attractor

Date: 2026-06-30
Status: public-safe attractor note

## Attractor

Pattern Custody

## Core phrase

A pattern may repeat without being allowed to testify.

## Meaning

Mirror Cartographer is strong because it preserves continuity. That same strength creates risk: continuity can seduce the system into treating recurrence as proof.

Pattern Custody turns recurrence into an inspectable object. It asks:

- who carried the pattern;
- what transformed it;
- what privacy boundary it crossed;
- what kind of claim it may support;
- what evidence is still missing;
- what user correction can revoke or downgrade it.

## Public-safe insight

The public artifact should not reveal the private path. It should reveal the custody structure that made the public abstraction safe.

## Relationship to previous attractors

- Boundary Stack: custody is one layer in the stack.
- Claim Routing: custody determines the route.
- Source Survivability: custody tests whether the source can carry the claim.
- Recall Condition: custody determines whether memory may enter.
- Public Index Covenant: custody determines what can be indexed.

## Use rule

Do not ask only whether a pattern is meaningful. Ask whether the pattern has custody.
