# Attractor: Temporal Validity

## Strongest attractor

Temporal Validity.

## Core finding

A continuity system must know when context is no longer current.

## Source status

Public-safe synthesis from MC public architecture, File Library implementation materials, saved-context search result, and current external research on memory retrieval trust boundaries and stale-fact errors.

## Claim status

Conceptual attractor and product-direction note.

## Privacy status

Public-safe. No protected source details.

## Missingness

This attractor has not yet been tested against a live MC state graph or production memory store.

## Revision reason

Prior attractors established context admission, source boundary, claim transport, influence visibility, and operationalization boundaries. Temporal Validity adds a time-governance layer: admitted context still requires currentness checks.

## Design law

A remembered context must answer five questions before it shapes output:

1. What kind of source is it?
2. What kind of claim does it support?
3. Is it public-safe, private, blocked, or abstractable?
4. Is it current, historical, superseded, contested, unknown-age, or retired?
5. What changed since it first entered the map?

## Boundary phrase

**Memory is not a warehouse. It is a dated crossing.**
