# Attractor: Compression Loss

## Name

Compression Loss

## Core claim

A safe summary should not pretend nothing was lost.

## Field behavior

When private or mixed context is transformed into a public artifact, the artifact gains safety, portability, and reviewability. It also loses specificity, emotional density, temporal sequence, contradiction texture, and source proximity.

The loss is not a failure if it is labeled. The failure is letting the polished public artifact impersonate the full source field.

## Boundary rule

Do not expose the source to prove the claim. Expose the loss boundary so the claim can be judged at the right strength.

## Product force

Compression Loss pulls MC toward:

- more honest public artifacts,
- weaker but safer claims,
- visible uncertainty,
- explicit missingness,
- reviewer-contestable methods,
- less aesthetic laundering of private material.

## Opposed failure mode

**False completeness**: the artifact is clean, fluent, and public-safe, but it hides the fact that the real source field was messier, more private, more contradictory, or less externally verified.

## Release mantra

Make the method visible. Make the loss visible. Keep the source protected.
