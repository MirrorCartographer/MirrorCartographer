# Attractor: Cross-Lane Contamination

Status: public-safe attractor note
Privacy status: no private content
Revision reason: names the force that this run found beneath claim routing and boundary stack work.

## Name

Cross-Lane Contamination

## Signal

The system connects many things correctly, then one connected thing begins to inherit proof it did not earn.

## Antidote

A proof-lane firewall.

## Operating law

Connection may travel. Authority must declare custody.

## Compression

Do not confuse connection with custody.

## What this adds to MC

The existing boundary stack asks whether claims are labeled. This attractor asks whether proof authority has leaked across labels.

## Design consequence

Every rich synthesis needs two simultaneous views:

- what connects
- what proves

If only the first view exists, the system becomes emotionally persuasive but epistemically unsafe. If only the second view exists, the system becomes safe but dead. MC needs both.
