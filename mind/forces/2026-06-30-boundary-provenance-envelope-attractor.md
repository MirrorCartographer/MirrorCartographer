# Boundary Provenance Envelope Attractor

## Strongest attractor
The next useful MC layer is not another interpretation engine. It is the object that decides whether an interpretation may travel.

## Core law
Meaning can connect. Evidence must stay in custody.

## Compression
A private pattern can shape a public requirement only if the private material is transformed into a boundary rule, not carried forward as content.

## Why this matters
Mirror Cartographer is valuable because it can preserve subtle meaning across time. The same ability creates risk: recurrence can feel like proof, coherence can feel like truth, and citation can look like safety even when the cited source cannot carry the claim.

The Boundary Provenance Envelope makes the system slower in the right place. It forces the export layer to ask:

1. What is this object?
2. Where did it come from?
3. What kind of claim is it making?
4. What can the source prove?
5. What must stay private?
6. What is missing?
7. Why did this version change?
8. What verification step comes next?

## Product implication
The public-facing MC demo should not only show a reflection. It should show the reflection's permission structure.

## Research implication
The next research path should compare user trust and comprehension with and without visible boundary/provenance envelopes.

## Implementation implication
The export pipeline should fail closed: if the envelope is incomplete, the object stays private, quarantined, or marked as needing more proof.

## Key phrase
Do not publish the path. Publish the custody rule that lets the insight survive without carrying the path.
