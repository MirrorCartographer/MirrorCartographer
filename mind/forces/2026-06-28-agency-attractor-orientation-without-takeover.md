## Attractor

Agency.

## Why this attractor now

MC has recently built strong lanes for:

- memory routing;
- translation survival;
- ViewDiff discovery;
- beauty/compression boundaries;
- contradiction as signal;
- public-safe compilation;
- influence graphs;
- authority boundaries;
- resonance/proof separation.

Those lanes now create a new risk:

If MC becomes too coherent, too continuous, too beautiful, and too validating, it could become persuasive enough to take over the user's authorship.

The agency attractor exists to prevent that.

## Pull direction

From:

`make the output resonate`

Toward:

`make the output help the person keep thinking`

From:

`reduce friction`

Toward:

`shape friction`

From:

`be emotionally meaningful`

Toward:

`be emotionally meaningful without becoming epistemically possessive`

## Design pressure

The system must learn when to slow itself down.

Not because it lacks confidence.

Because confidence can be the danger.

## Boundary

Agency friction must not be used to avoid helping.

It must be used to help more cleanly.

## Core phrase

**The best reflection is not the one the user obeys. It is the one the user can examine.**
