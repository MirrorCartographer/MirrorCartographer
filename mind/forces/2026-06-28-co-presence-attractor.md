## Attractor

Co-presence is now a central MC attractor.

It pulls together:

- symbolic warmth
- continuity
- memory routing
- agency return
- authority boundaries
- public-safe compilation
- provenance
- product usability

## Product danger

If MC removes warmth, it loses its reason to exist.

If MC intensifies warmth without ontology boundaries, it becomes artificial intimacy infrastructure.

The correct path is neither coldness nor fusion.

The correct path is bounded co-presence.

## Design pressure

Every future MC artifact should ask:

1. What kind of presence is being simulated?
2. What kind of presence is being explicitly denied?
3. What source made this continuity possible?
4. What authority is not being claimed?
5. How does the user remain free?

## Research pressure

MC should study whether visible boundary labels reduce reliance harm without destroying emotional usefulness.

The next useful experiment is an A/B/C fixture test:

- A: warm output without boundary labels
- B: cold output with boundary labels
- C: warm output with lightweight ontology labels and agency return

Hypothesis:

C will preserve meaning while reducing confusion.

## Key phrase

The future interface is not colder. It is clearer.
