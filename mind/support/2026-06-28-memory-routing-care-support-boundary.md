# Care Support Boundary — Memory Routing

Status labels

- Source status: derived from MC public-safe care-support lane and memory-routing hypothesis.
- Claim status: support-design proposal, not medical, clinical, diagnostic, or treatment guidance.
- Privacy status: public-safe; no personal, household, health, animal-care, or private details.
- Missingness: not reviewed by clinicians, caregivers, or compliance experts.
- Revision reason: created to keep the evidence-based medical/social-care lane active without crossing into private or diagnostic content.

## Boundary statement

Memory routing may be useful in care-adjacent contexts only when it remains non-diagnostic and source-aware.

The system may help organize observations, questions, timelines, and handoff packets.

It must not claim disease, treatment, causality, prognosis, or medical authority.

## Public-safe care-support hypothesis

A care-support AI should not merely summarize previous observations.

It should route observations according to audience and authority.

Example routes:

- private reflection route
- caregiver route
- professional handoff route
- appointment-prep route
- public-safe research route
- blocked route

## Why routing matters

The same observation can be useful in one context and unsafe or misleading in another.

A private note may help a person remember what happened.

A professional handoff may need dates, frequencies, uncertainties, and questions.

A public artifact may need only the abstracted method.

A medical answer may need to say: this is not enough to conclude.

## Required labels for care-adjacent memory

- source status
- observation vs interpretation
- user-reported vs professionally measured
- current vs historical
- unresolved vs confirmed
- non-diagnostic boundary
- missing data
- urgency flag if applicable

## Prohibited moves

- diagnosing
- implying certainty from patterns alone
- using symbolic interpretation as clinical evidence
- converting private observations into public claims
- hiding uncertainty
- presenting speculation as care guidance

## Useful public-safe product requirement

Create a Care Route Card:

- What was observed?
- Who can this be shown to?
- What should be abstracted?
- What should be blocked?
- What question does it raise?
- What evidence is missing?
- What professional review would be appropriate?

## Key phrase

Care memory should not become authority. It should become better handoff.
