# Force: Legibility Is a Safety Surface

## Attractor

Beauty

## Force statement

Legibility is not cosmetic. It is a safety surface.

When governance artifacts become too dense, fragmented, or visually/semantically hostile, accountability fails at the point of use. The system may contain the right fields, but the human cannot perceive the right boundary in time.

## Pressure

The GitHub Mind is accumulating strong primitives:

- trace,
- sufficiency,
- intervention boundary,
- exit authority,
- review loop,
- handoff integrity,
- context lease,
- claim portability.

As the primitive set grows, a new failure mode appears: governance complexity can itself become a hazard.

## Counter-force

Aesthetic structure can prevent collapse by making important distinctions visible:

- observation vs interpretation,
- evidence vs claim,
- documentation vs decision support,
- confidence vs authority,
- permission vs continuation,
- privacy-safe vs private,
- current vs expired context,
- supported vs portable.

## Repository implication

Every new governance primitive should eventually receive a human-facing form:

- short definition,
- plain-language reading,
- action ceiling,
- failure mode,
- review trigger,
- source basis,
- missingness.

## Labels

- Source: Synthesis from current repository trajectory and public governance/care documentation signals.
- Claim: Legibility is a safety surface because unreadable governance cannot reliably govern action.
- Privacy: Public-safe.
- Missingness: Needs implementation as templates or cards.
- Revision: Initial force note.
- Confidence: Medium-high.
- Evidence maturity: Moderate.
- Action authority: Repository-architecture guidance only.
- Validation status: Unvalidated.
