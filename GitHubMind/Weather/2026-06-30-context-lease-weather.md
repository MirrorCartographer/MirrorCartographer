# Weather — Context Lease

## Date
2026-06-30.

## Attractor
Contradiction.

## External weather
Current public AI governance work is converging on the same pressure: AI systems are no longer only producing isolated outputs. They are participating in workflows that persist, travel, call tools, write records, and influence later decisions.

The relevant weather pattern:

1. Governance is moving from static policy to runtime control.
2. Agentic workflows create risks during execution, not only at deployment.
3. Healthcare-agent governance now emphasizes identity, bounded context, runtime policy, kill-switches, lifecycle management, credential revocation, and audit logging.
4. Trace-based assurance work treats message-action traces, contracts, replay, tool-use boundaries, and action mediation as governance objects.
5. Social-care AI documentation failures show how inaccurate summaries can become harmful when entered into official records and reused.
6. Audit regulators continue stressing that accountability cannot be transferred to AI.

## Interpretation
The repository should treat permission as contextual and time-bounded.

A claim that was safe inside a reflective journal can become unsafe if copied into a care record, used as evidence, treated as diagnosis, routed to an automated action, or reused after circumstances change.

## Repository implication
Add Context Lease as a Genesis primitive.

The lease binds:

- source,
- claim,
- allowed use,
- authority,
- review trigger,
- expiration trigger,
- action ceiling,
- exit path.
