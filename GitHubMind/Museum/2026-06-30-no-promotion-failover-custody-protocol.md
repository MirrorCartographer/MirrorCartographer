## Candidate artifact

**Failover Custody Protocol**

## Why it matters

This primitive fills a real gap in the current GitHub Mind: interruption and recovery need a named custody pathway. A stopped system can still leave obligations behind.

## Why it is not promoted

The primitive is strong, but it has not yet been tested against:

- a concrete Mirror Cartographer workflow;
- an income-facing review template;
- a care-continuity example;
- a failure drill;
- a governance-kernel state machine.

## Promotion requirements

Promote only after at least two of the following exist:

1. A worked example showing normal custody → interrupted custody → fallback custody → returned custody.
2. A public-safe consulting checklist for AI Failover Custody Review.
3. A Care Custody Continuity Map template.
4. A Governance Kernel state diagram.
5. A recovery drill that includes fallback custodian activation and return conditions.
